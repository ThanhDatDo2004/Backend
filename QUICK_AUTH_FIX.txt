❌ ERROR:
   GET /api/shops/me 401
   GET /api/shops/me/bookings 401

✅ REASON:
   Missing Bearer token in Authorization header

─────────────────────────────────────────────────────────────────────────────

🔧 3-STEP FIX:

Step 1: Get Token
────────────────
POST http://localhost:5050/api/auth/login

Body:
{
  "email": "shop@example.com",
  "password": "password123"
}

Response will show:
{
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}

Step 2: Copy Token
──────────────────
Copy the long token string

Step 3: Add to Postman
──────────────────────
Option A - Per Request:
  1. Go to Headers tab
  2. Add header:
     Key: Authorization
     Value: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Option B - Entire Collection:
  1. Right-click collection
  2. Edit
  3. Go to Authorization tab
  4. Type: Bearer Token
  5. Token: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
  6. Save

─────────────────────────────────────────────────────────────────────────────

✅ TEST:
   Now retry:
   GET /api/shops/me
   
   Should return 200 OK ✅

─────────────────────────────────────────────────────────────────────────────

⚡ POSTMAN TIP:
   
   In Postman, after login:
   1. Response → Copy token
   2. Collection → Authorization → Bearer Token
   3. Paste token
   4. All requests now authenticated! ✅

─────────────────────────────────────────────────────────────────────────────

🎯 REMEMBER:
   • Token required for ALL endpoints with /shops/me or auth
   • Format: Bearer XXXXX (must include "Bearer ")
   • If test fails, get new token
   • Check Authorization header in Postman Console (Ctrl+Alt+C)

─────────────────────────────────────────────────────────────────────────────
